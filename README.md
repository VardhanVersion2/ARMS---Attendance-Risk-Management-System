## 🚀 Overview

**ARMS** is a predictive optimization toolkit engineered to help college students dynamically manage, simulate, and safeguard their academic standing. Built to solve the real-world anxieties surrounding strict institutional **75% attendance criteria**, the application reverse-engineers college ERP calculation rules into a highly responsive, user-centric dashboard.

Instead of guessing manual records, ARMS provides immediate statistical forecasting, ensuring students stay compliant, avoid administrative penalties, and maximize their schedule flexibility.

👉 **Live Production Build:** [attenguide.netlify.app](https://attenguide.netlify.app)

## ✨ Core Architecture & Features

### 🎯 ERP-Grade Precision Alignment
Simulates the exact internal algorithms used by college ERP tracking systems to guarantee calculation metrics sync seamlessly down to the decimal point.

### 📉 Smart Bunk Margin Calculator
Dynamically computes the precise number of lecture hours a student can safely skip ("bunks") consecutively without violating the hard 75% threshold constraint.

### 💰 Financial Risk Forecasting
Proactively monitors attendance data to compute and predict potential financial penalties or mandatory fine thresholds before they incur.

### ⚠️ Proactive Risk Isolation
Runs continuous evaluation models on existing inputs to instantly flag low-attendance risks, categorizing student profiles into actionable warning tiers.

## 🛠️ Technical Matrix

The project is structured as a decoupled Minimum Viable Product (MVP) prioritizing rapid runtime responsiveness and local storage processing speed.

* **Frontend Engine:** React.js / Modern UI Architecture
* **Core Languages:** JavaScript ($93.5\%$), HTML5 ($6.5\%$), CSS3
* **Repository Layout:**
  * `/ARMS-MVP` — Core computational logic, business algorithms, and server foundations.
  * `/Frontend` — Client interface layers and state management configurations.
